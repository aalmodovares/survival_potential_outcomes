# Survival Potential Outcomes

Code accompanying a paper on treatment effect estimation with survival outcomes under censoring. The repository contains:

- Model implementations in [models](/C:/Users/Alex/CODE/survival_potential_outcomes/models)
- Experiment entry points in [evaluation](/C:/Users/Alex/CODE/survival_potential_outcomes/evaluation)
- Public IHDP source data and the IHDP survival data generator in [data](/C:/Users/Alex/CODE/survival_potential_outcomes/data)
- Analysis notebooks for figures, tables, and post-processing

The two main experiment scripts are:

- [evaluation/main_ihdp.py](/C:/Users/Alex/CODE/survival_potential_outcomes/evaluation/main_ihdp.py)
- [evaluation/main_CRC_censoring.py](/C:/Users/Alex/CODE/survival_potential_outcomes/evaluation/main_CRC_censoring.py)

## Highlights

- Reproducible semisynthetic IHDP survival benchmark
- Real-world colorectal cancer use case with censoring-aware preprocessing
- Multiple baselines enabled by default in the main experiment scripts
- Notebooks kept in the repository for result analysis, tables, and plots

## Environment Setup

Create and activate a Python environment, then install the dependencies:

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Some dependencies are heavy and platform-sensitive. A clean virtual environment is strongly recommended.

## Data

### IHDP

The public IHDP source data used by the experiments is included under:

```text
data/IHDP/IHDP
data/IHDP/IHDP_b
```

The IHDP survival datasets used by `evaluation/main_ihdp.py` are generated locally and are intentionally not versioned. Generate them with:

```bash
python data/IHDP/ihdp_survival.py
```

This creates:

```text
data/IHDP/IHDP_survival_a
data/IHDP/IHDP_survival_b
data/IHDP/IHDP_survival_c
```

### Colorectal Cancer

The Colorectal Cancer use case uses the original `a_endpt.sas7bdat` dataset file. This dataset is not included in the repository because it cannot be made public. Obtain the original file from the authorized dataset source and place it at:

```text
data/a_endpt.sas7bdat
```

Then run [evaluation/preprocess_crc_data.ipynb](/C:/Users/Alex/CODE/survival_potential_outcomes/evaluation/preprocess_crc_data.ipynb) to generate the local preprocessed file used by the CRC training script:

```text
data/use_cases/preprocessed_crc_data.pkl
```

Generated CRC preprocessed data, experiment results, plots, logs, and checkpoints are not included in the repository.

## Running the Experiments

### 1. IHDP Survival Benchmark

First generate the local survival variants:

```bash
python data/IHDP/ihdp_survival.py
```

Then run the benchmark:

```bash
python evaluation/main_ihdp.py
```

The script trains and evaluates the enabled models across the three survival settings:

- `_survival_a`
- `_survival_b`
- `_survival_c`

By default, the paper baselines are enabled in the script:

- `s-learner`
- `t-learner`
- `tarnet`
- `cfrnet`
- `sa-tedvae-m0`
- `sa-tedvae-m1`
- `sa-tedvae-m2`

Outputs are written locally to:

```text
evaluation/results_IHDP/
```

### 2. CRC Censoring Use Case

First place the restricted source file at:

```text
data/a_endpt.sas7bdat
```

Then open and run:

```text
evaluation/preprocess_crc_data.ipynb
```

This creates:

```text
data/use_cases/preprocessed_crc_data.pkl
```

After preprocessing, run:

```bash
python evaluation/main_CRC_censoring.py
```

The script evaluates the enabled models under:

- `no_censor`
- `ic`

Outputs are written locally to:

```text
evaluation/results_CRC_censoring/
```

- [evaluation/crc_censoring_plots.ipynb](/C:/Users/Alex/CODE/survival_potential_outcomes/evaluation/crc_censoring_plots.ipynb): CRC figures, comparison plots, summary tables, and individual-patient visualization

Run the notebooks only after generating the corresponding local result folders.

## What Is Intentionally Not Versioned

The repository does not track:

- CRC source data
- CRC preprocessed data
- Generated IHDP survival datasets
- Experiment result folders
- Plots produced from local runs
- Model checkpoints
- Local caches and IDE artifacts

These paths are already covered in `.gitignore`.
